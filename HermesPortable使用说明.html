它是什么

HermesPortable 是 Hermes Agent（Nous Research 出品）的"开箱即用"版本。

它把 AI 助手、Python 运行时、依赖库，和一个图形化配置页面全部打包在一个文件夹里。你什么都不用装，下载、解压、双击，就能在 macOS、Linux、Windows 上直接运行。

✨
为什么叫"便携版"？解压到 U 盘，插到任何机器上都能直接用。关闭之后所有数据留在 U 盘里，宿主机不留一个字节。

能做什么

功能说明
智能对话多行输入、流式回复、斜杠命令
接入聊天软件Telegram / Discord / Slack / WhatsApp / Signal / 邮件
持久记忆自动记录关键信息，跨会话保留上下文
内置工具40+ 工具：网页浏览、搜索、文件操作、语音、定时任务…
模型供应商支持 15+ 家（OpenRouter、Anthropic、OpenAI、Google、DeepSeek 等）
子代理分身协作、并行处理任务

解压

把下载的 zip 文件解压到一个方便访问的位置。推荐的位置：

系统推荐位置
WindowsU 盘根目录，或 D:\HermesPortable
macOS"应用程序" 文件夹，或桌面
Linux用户主目录 ~/HermesPortable
⚠
Windows 用户请注意：
  • 不要放在路径太深的目录（Windows 有 260 字符路径限制）
  • 不要放在中文路径里（比如 桌面 在系统内部是中文路径）—— 优先放到 C:\HermesPortable 或 U 盘根目录
  • 不要放在 OneDrive / 坚果云 / iCloud Drive 等同步盘里（同步会破坏文件结构）
💡
Universal 是什么？三合一包把三个系统的运行环境全部塞进一个 zip。适合想把 U 盘插到不同机器（一会 Mac、一会 Windows）共用的场景。如果你只在一台机器上用，下单平台版就够了，省空间。
🚀

首次启动（三步上手）

解压完成后，找到对应平台的启动器，双击它。你不需要打开终端、不需要输任何命令。

🖥
Windows 双击 Hermes.bat
🍎
macOS 双击 Hermes.command
🐧
Linux 双击 Hermes.sh 或在终端运行

接下来会发生什么

1 启动器弹出一个黑色窗口，开始做一次性准备工作（大约 10 秒）。这期间请不要关闭它。
2 浏览器自动打开一个配置页面（地址 http://127.0.0.1:17520）。在这里你需要填入一个"API 钥匙"（API Key）—— 它是 Hermes 跟 AI 模型对话的通行证。
不知道从哪里拿 API Key？直接跳到下面的 免费模型配置，里面有免费方案。最推荐用 OpenRouter，注册即可获取免费 Key。
3 在配置页面填好 Key 之后，点 "Save & Start"（保存并启动）按钮。几秒后，对话窗口就在黑色终端里启动了，开始聊天吧。
🌐
程序会占用的端口：
配置页面 17520 对话查看 17521 Web UI 8648 消息网关 8642
只在本机监听（127.0.0.1），外部访问不到。
🔐
零痕迹保证：启动器会在便携包目录里创建一个 _home/ 沙箱，所有读写都在 U 盘里进行，宿主机的"我的文档"、配置文件夹完全不会被碰。拔掉 U 盘，什么都带走、什么都没留下。

以后再次启动

下次用的时候，直接双击同一个启动器就行。Hermes 会读取上次保存的配置，跳过浏览器配置页面，直接进入对话。

更新与备份

更新到新版本（推荐：下载新 zip）

有新版本发布时，最简单的方法就是：

1从发布页下载新 zip
2先把旧的 data/ 文件夹复制出来备份（这里面有你的配置、API Key、对话历史、技能等所有个人数据）
3删掉旧文件夹，解压新 zip
4把备份的 data/ 文件夹复制回新文件夹，覆盖新 zip 里的 data/
5双击启动器，一切照旧

只更新内部 AI 引擎

如果只是想把核心引擎 hermes-agent 升级到最新，而不换整个便携包，可以用：

hermes update

等同于在终端里运行 python update.py update。

备份数据

💾
所有个人数据（API Key、对话、记忆、配置）都在 data/ 文件夹里。直接复制这个文件夹就是完整备份。想恢复就把它放回去。

免费模型配置

以下方案均无需付费即可使用 AI 模型，按推荐程度排序。第一次用的话，强烈建议从方案一开始。

方案一：OpenRouter（最推荐）

注册账号就给免费 Key，支持多个免费模型，最适合新手。

1前往 openrouter.ai/keys 注册账号（支持 Google / GitHub 登录）
2点 "Create Key" 创建一个 API Key，复制
3在 Hermes 的 Web 配置页面（http://127.0.0.1:17520）的 "OpenRouter" 字段粘贴 Key，点保存即可。
4或者手动编辑 data/.env，添加一行：
OPENROUTER_API_KEY=sk-or-v1-你的密钥

nvidia/nemotron-3-super-120b-a12b:freeNVIDIA Nemotron
arcee-ai/trinity-large-preview:freeArcee Trinity
💡
OpenRouter 经常上新免费模型。在 Hermes 里运行 hermes model，选 OpenRouter 后就能看到当前所有免费模型。

方案二：Google Gemini 免费层

Google 提供免费额度，模型质量高，但有请求频率限制（短时间请求过多会被限流）。

方式 A：API Key

1去 Google AI Studio 创建免费 API Key
2编辑 data/.env：GOOGLE_API_KEY=你的密钥
3运行 hermes config set model.default "gemini/gemini-3-flash-preview"

方式 B：浏览器登录，免 API Key

1运行 hermes model，选 google-gemini-cli 供应商
2按提示用浏览器完成 Google 登录，不用 API Key

方案三：Nous Portal 免费层

2登录后自动检测免费层

常见问题

启动后黑窗口一闪而过就关了？
通常是解压位置问题。请检查：
  • 路径里有没有中文字符
  • 路径是否太深（Windows 有 260 字符限制）
  • 是否放在 OneDrive 等同步盘里（同步会破坏文件）
把整个文件夹挪到 C:\HermesPortable 或 U 盘根目录再试。
启动后浏览器没有自动打开？
手动在浏览器打开 http://127.0.0.1:17520，或在终端里运行 hermes setup。
提示找不到 Python 或者 hermes_cli 模块？
这是旧版本 v0.13.6 / v0.13.7 的已知问题。请从 最新发布页下载 v0.13.8 或更新的 zip 包，已修复。
macOS 提示"无法打开，因为 Apple 无法检查其是否有恶意软件"？
右键点击 Hermes.command，选"打开"（不是双击），在弹窗里再点一次"打开"。只需要这样做一次，以后就可以双击了。或者在系统设置 → 隐私与安全性 里放行。
Linux 下双击 Hermes.sh 没反应？
大部分 Linux 文件管理器默认不会双击执行脚本。打开终端，cd 到解压目录，然后运行 ./Hermes.sh。 如果提示权限不够，先 chmod +x Hermes.sh。
API Key 配置在哪？
最简单的办法是启动后自动打开的 Web 配置页面 http://127.0.0.1:17520。也可以手动编辑 data/.env，例如添加 ANTHROPIC_API_KEY=sk-xxx。
配置好了但 AI 不回话 / 报错？
运行 hermes doctor 自动诊断。如果还有问题，查看 data/logs/ 里最新的日志文件，错误信息会写在里面。
想完全重置怎么办？
删掉整个 data/ 文件夹，然后重新启动。会丢失全部对话历史、技能和 Key，建议删除前备份。
能换到另一台电脑继续用吗？
能。这就是便携版的意义——整个文件夹复制到 U 盘或新机器，双击启动器照样运行，所有对话、记忆、Key 都在。跨系统（如从 Mac 拷到 Windows）则需要下对应平台的 zip 或 Universal 三合一版。
