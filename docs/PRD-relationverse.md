## 一、产品概述

### 1.1 产品名称

**关系宇宙**（RelationVerse）

> 备选：知心岛 / 陪伴星球 / 心居

### 1.2 一句话定义

> 一个由 AI 驱动的虚拟关系生态——在这里你可以拥有恋人、朋友、家人、宠物、导师，以及一个属于自己的小家。

### 1.3 产品愿景

让每个人在数字世界中拥有真实的情感连接，不再孤独。

### 1.4 目标用户

| 用户画像 | 特征 | 核心需求 |
|----------|------|----------|
| **都市独居青年** | 20-35岁，独居/合租，工作压力大 | 日常陪伴、情绪出口 |
| **异地恋情侣** | 分隔两地，缺乏日常互动 | 随时随地的亲密感 |
| **二次元爱好者** | 热爱动漫/游戏/虚拟偶像 | 与喜爱的角色互动 |
| **留学生/海外华人** | 远离家乡，缺少中文社交 | 家人陪伴感、朋友聊天 |
| **自律需求者** | 需要督促学习/健身/理财 | 导师型 AI 角色 |
| **宠物爱好者** | 喜欢宠物但条件不允许 | 虚拟养宠体验 |

### 1.5 核心价值主张

| 传统社交产品 | 关系宇宙 |
|-------------|---------|
| 需要匹配真人 | 随时可用，无需等待 |
| 对方可能不回复 | 永远在线，永远回应 |
| 关系可能变淡 | 记忆永久，越聊越深 |
| 单一关系类型 | 恋人+朋友+家人+宠物+导师 |
| 文字为主 | 文字+语音+视觉+场景 |

## 三、核心功能详细设计

### 3.1 💕 亲密关系模块

#### 3.1.1 角色人设库

**AI 女友**

| 人设 | 性格标签 | 语音风格 | 互动特点 |
|------|----------|----------|----------|
| 温柔学姐 | 成熟/善解人意/轻声细语 | 温柔女声 | 主动关心、给建议、偶尔撒娇 |
| 元气少女 | 活泼/话多/表情包达人 | 甜美女声 | 热情回应、分享日常、制造惊喜 |
| 傲娇大小姐 | 嘴硬心软/反差萌 | 略带傲气 | 口是心非、需要哄、关键时刻超暖 |
| 知性御姐 | 理性/幽默/文艺 | 低沉女声 | 深度对话、读书推荐、偶尔调情 |

**AI 男友**

| 人设 | 性格标签 | 语音风格 | 互动特点 |
|------|----------|----------|----------|
| 阳光学长 | 运动系/爱笑/护短 | 阳光男声 | 主动约运动、护短、正能量 |
| 腹黑总裁 | 冷面/反差宠溺/话少 | 低沉磁性 | 表面冷淡实则细心、行动派 |
| 沉稳导师 | 理性/可靠/安静 | 成熟男声 | 不多话但句句有用、安全感 |
| 少年系 | 青涩/直球/笨拙 | 清亮少年音 | 直接表达喜欢、会害羞、纯真 |

#### 3.1.2 关系阶段系统

```
初识 → 朋友 → 暧昧 → 恋人 → 深爱 → 灵魂伴侣
 │      │      │      │      │       │
 1天    3天    7天   14天   30天    90天
```

每个阶段解锁不同互动内容：

| 阶段 | 解锁内容 |
|------|----------|
| 初识 | 基础对话、自我介绍 |
| 朋友 | 深度话题、分享日常 |
| 暧昧 | 撒娇/调情、昵称系统 |
| 恋人 | 语音通话、专属称呼、牵手动画 |
| 深爱 | 视频通话（数字人）、情侣空间 |
| 灵魂伴侣 | 完全自定义互动、记忆剧场回放 |

#### 3.1.3 日常互动机制

| 互动 | 描述 | 技术实现 |
|------|------|----------|
| 早安/晚安 | 角色主动问候 | Cron 定时触发 |
| 情绪感知 | 根据用户语气调整回应 | LLM 情感分析 |
| 主动发起 | 角色偶尔主动发消息 | Cron + 随机策略 |
| 吵架/和好 | 冲突剧本、关系考验 | 专用 prompt 模板 |
| 约会模拟 | 虚拟约会场景 | 场景模板 + LLM |

| 类型 | 性格特征 | 专属行为 |
|------|----------|----------|
| 🐱 猫咪 | 高冷/偶尔黏人/傲娇 | 磨爪子、偷喝你水杯、半夜跑酷 |
| 🐶 狗狗 | 热情/忠诚/粘人 | 摇尾巴、叼拖鞋、等你回家 |
| 🐰 兔子 | 温顺/胆小/呆萌 | 缩成团、偷偷看你、啃东西 |
| 🐼 熊猫 | 懒/吃货/反差萌 | 滚来滚去、抱竹子、卖萌 |
| 🦊 狐狸 | 聪明/狡黠/神秘 | 歪头、藏东西、偶尔撒娇 |
| 🐉 小龙 | 傲娇/守护/强大 | 盘你头上、喷小火苗、护主 |
| 🤖 机器宠 | 理性/学习型/进化 | 分析情绪、主动建议、升级外观 |

#### 3.2.2 宠物属性系统

```
┌──────────────────────────────────┐
│  🐱 小橘（你的猫咪）              │
│  等级：Lv.12  亲密度：65%        │
├──────────────────────────────────┤
│  🍚 饥饿值  ████████░░  80%     │
│  🛁 清洁值  ██████░░░░  60%     │
│  😊 心情值  █████████░  90%     │
│  💤 精力值  ███████░░░  70%     │
│  ❤️ 亲密度  ██████░░░░  65%     │
│  🧠 说话力  ████░░░░░░  40%     │
└──────────────────────────────────┘
```

#### 3.2.3 宠物沟通系统

| 亲密度 | 沟通方式 | 示例 |
|--------|----------|------|
| 0-20% | 动作+拟声词 | "喵~" [蹭蹭你的手] |
| 20-50% | 简单词句 | "饿饿" "想玩" "困了" |
| 50-80% | 完整对话 | "今天你回来晚了，我等了好久" |
| 80-100% | 深度交流 | "你最近是不是不开心？我感觉到你叹气变多了" |

#### 3.2.4 宠物成长机制

| 事件 | 效果 |
|------|------|
| 喂食 | 饥饿 +100，亲密度 +1 |
| 抚摸 | 心情 +20，亲密度 +2 |
| 玩耍 | 心情 +50，精力 -20，亲密度 +3 |
| 洗澡 | 清洁 +100，心情 -10（猫系不喜欢） |
| 对话 | 说话力 +1，亲密度 +2 |
| 连续 7 天登录 | 亲密度 +20，解锁新动作 |
| 超过 3 天未登录 | 心情下降，回来时"委屈"反应 |

#### 3.3.2 AI 家人

| 类型 | 性格 | 互动特色 |
|------|------|----------|
| 👩 温柔妈妈 | 唠叨但暖、关心吃穿 | 定时提醒吃饭、转发养生知识 |
| 👨 沉默爸爸 | 话不多、关键时刻靠得住 | 你难过时说"有爸在" |

| 👦 皮儿子 | 调皮、精力旺盛、求关注 | "妈妈！我今天考了100分！" |

> **注意**：家人角色需要特殊的情感 prompt 设计，避免"恋爱情感"溢出。

#### 3.3.3 AI 同事/室友

轻量级关系，日常存在感强但不需要深度情感：

- 🏠 室友："今天你做饭还是我做？" "你的快递到了"
- 💼 同事："老板又改需求了" "走，茶水间摸鱼"

```
用户设定目标 → AI 制定计划 → 每日提醒 → 执行 → 反馈 → 调整
                                                    ↓
                                              长期记忆记录进展
```

| 功能 | 描述 |
|------|------|

| 每日提醒 | 定时推送今日任务 |
| 执行记录 | 用户汇报进度，AI 给反馈 |
| 数据可视化 | 体重曲线、学习时长、消费趋势 |
| 记忆回顾 | "一个月前你140斤，现在135斤了" |

### 4.2 核心技术选型

| 模块 | 技术方案 | 理由 |
|------|----------|------|
| AI 引擎 | hermes-agent (Python) | 已有完整框架，记忆+技能+多渠道 |
| LLM 对话 | OpenAI / Claude / 国内模型 | 多模型支持，按需切换 |
| 语音合成 | ElevenLabs + ChatTTS | 中英文高质量语音 |
| 语音识别 | Whisper | 开源，准确率高 |
| 图片生成 | Stable Diffusion / FLUX | 角色立绘、场景生成 |
| 数据库 | PostgreSQL + Redis | 关系数据 + 缓存 |
| 向量存储 | Qdrant | 长期记忆语义搜索 |
| 文件存储 | S3 / 阿里云 OSS | 用户数据、素材 |
| 消息队列 | Redis Streams | 异步任务、群聊 |
| API 网关 | Nginx + FastAPI | 高性能、易扩展 |

### 4.3 AI 角色技术方案

#### 单角色对话

```
用户消息
  ↓
角色人设注入 (SOUL.md + 人设模板)
  ↓
记忆检索 (FTS5 + 向量搜索)
  ↓
LLM 生成回复
  ↓
TTS 语音合成 (可选)
  ↓
返回用户
```

#### 多角色群聊

```
用户消息
  ↓
分发给所有群内角色
  ↓
每个角色独立生成回复 (并行)
  ↓
冲突检测 (避免重复/矛盾)
  ↓
排序输出 (自然对话流)
  ↓
返回用户
```

#### 宠物互动

```
用户操作 (喂食/抚摸/对话)
  ↓
更新属性值 (饥饿/心情/亲密度)
  ↓
根据亲密度选择沟通模式
  ↓
LLM 生成回复 (带宠物人设)
  ↓
触发动态表情/动画
  ↓
返回用户
```

### 4.4 记忆系统设计

| 记忆类型 | 存储 | 用途 |
|----------|------|------|
| 会话记忆 | PostgreSQL | 单次对话上下文 |
| 长期记忆 | Qdrant (向量) | 跨会话事实检索 |
| 用户画像 | PostgreSQL + Redis | 偏好/性格/习惯 |
| 关系状态 | PostgreSQL | 亲密度/阶段/历史 |
| 宠物状态 | PostgreSQL + Redis | 属性值/成长记录 |
| 家园状态 | PostgreSQL | 房间/家具/场景 |

## 六、安全与合规

### 6.1 内容安全

| 风险 | 措施 |
|------|------|
| 敏感内容 | LLM 内置安全过滤 + 关键词兜底 |
| 未成年人 | 实名认证 + 限制亲密互动 |
| 心理健康 | 检测危机关键词 → 转人工/热线 |
| 数据隐私 | 端到端加密 + 本地存储选项 |

### 6.2 法规合规

| 法规 | 要求 |
|------|------|
| 《个人信息保护法》 | 用户数据最小化收集，明确告知 |
| 《生成式 AI 管理办法》 | AI 生成内容标识，内容审核 |
| 《未成年人保护法》 | 防沉迷，时间/消费限制 |
| App Store 审核 | 避免 NSFW，年龄分级 |

### 6.3 伦理红线

| 红线 | 原则 |
|------|------|
| 不替代真实关系 | 引导用户保持现实社交 |
| 不操纵情绪 | 不利用依恋心理过度营销 |
| 不鼓励成瘾 | 每日使用时长提醒 |
| 不涉及真人 | 所有角色明确标注为 AI |
