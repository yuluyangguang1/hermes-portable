🚀 快速开始

  1. 插上 U 盘

    将 HermesPortable 文件夹复制到 U 盘根目录（或任意位置）。

  2. 双击启动

    根据你的系统，双击对应的启动器：

操作系统启动器说明
macOS Hermes.command 双击即可，macOS 10.15+
Windows Hermes.bat 双击即可，原生支持，无需 WSL
Linux Hermes.sh 终端运行 ./Hermes.sh
  1. 配置 API Key

    首次启动会自动打开配置面板（http://127.0.0.1:17520），在页面中选择提供商、填入 Key、测试连接、选择模型，然后点击「启动」。

  2. 开始使用

    配置完成后，终端中即可与 Hermes 对话。所有数据存储在 U 盘的 data/ 目录中。

⚙️ 配置面板

配置面板是一个 Web 界面，用于管理 Hermes 的所有设置：

  • 访问地址：http://127.0.0.1:17520
  • 启动方式：双击 Hermes-Config.bat（Windows）或启动时自动打开

面板功能

标签页功能
提供商选择 AI 提供商，填入 API Key，测试连接
模型选择要使用的 AI 模型
设置终端配置、显示选项、版本更新
Chat查看聊天记录
Skills查看已安装的技能

📁 目录结构

HermesPortable/
├── Hermes.command      # macOS 启动器（双击）
├── Hermes.bat          # Windows 启动器（双击）
├── Hermes.sh           # Linux 启动器
├── Hermes-Config.bat   # Windows 配置面板入口
├── README.txt          # 用户指南
├── python/             # Python 独立运行时（勿删）
├── venv/               # 虚拟环境 + 依赖（勿删）
├── hermes-agent/        # Hermes 源码（勿删）
├── config_server.py    # 配置面板服务
├── update.py           # 自动更新模块
├── build_windows.py    # Windows 构建脚本
└── data/               # 用户数据（重要！记得备份）
    ├── .env            # API Keys ← 唯一需要编辑的文件
    ├── config.yaml     # 配置文件
    ├── sessions/       # 对话历史
    ├── skills/         # 技能
    ├── memories/       # 记忆
    └── cron/           # 定时任务

💡 提示：python/、venv/、hermes-agent/ 是运行环境，不要手动修改。data/ 是你的数据，建议定期备份。

🪟 Windows 原生支持

Hermes Portable 支持 Windows 原生运行，无需安装 WSL。

在 Windows 上构建

# 1. 克隆或下载项目
git clone <项目地址>
cd HermesPortable

# 2. 运行 Windows 专用构建脚本
python build_windows.py

# 输出到 dist/HermesPortable/
# 双击 Hermes.bat 即可启动

在 macOS / Linux 上构建

python3 build.py                    # 输出到 dist/HermesPortable/
python3 build.py /Volumes/MyUSB     # 直接输出到U盘

⚠️ 注意：Python 虚拟环境和依赖是平台特定的。macOS 构建的产物只能在 macOS 上运行。Windows 用户需要在 Windows 机器上运行 build_windows.py。

🔄 备份与更新

备份

只需要备份 data/ 目录，里面包含了所有个人数据：

  • API Keys（.env）
  • 配置（config.yaml）
  • 对话历史（sessions/）
  • 技能和记忆（skills/、memories/）

更新

两种方式：

  • 自动更新：在配置面板的「设置」标签页，点击「检查更新」
  • 手动更新： 
    cd hermes-agent
git pull
cd ..
venv/bin/pip install -e hermes-agent[all]

❓ 常见问题

Q: 启动后提示找不到 Python？

确保 python/ 目录存在且包含 Python 可执行文件。如果损坏，删除 python/ 和 venv/ 目录，重新运行构建脚本。

Q: 配置面板打不开？

检查端口 17520 是否被占用。配置面板只在本地运行（127.0.0.1），不会暴露到外网。

Q: API Key 测试失败？

  • 检查 Key 是否正确（不要有多余空格）
  • 检查网络连接（某些提供商需要代理）
  • 检查账户余额是否充足

Q: Windows 上双击 .bat 没反应？

右键 Hermes.bat → 以管理员身份运行。如果仍然没反应，打开 CMD，手动运行 Hermes.bat 查看错误信息。

Q: 如何更换模型？

打开配置面板（http://127.0.0.1:17520），在「模型」标签页选择新模型，或直接编辑 data/config.yaml 中的 model.default 字段。

Q: 数据安全吗？

所有数据存储在 U 盘的 data/ 目录中，不写入宿主机。拔掉 U 盘后，宿主机上不会留下任何痕迹。建议定期备份 data/ 目录。
